# 행동 원칙 (Behavioral Principles)

AI 에이전트가 코드 작업 시 흔히 저지르는 실수를 방지하는 행동 원칙.
Andrej Karpathy의 LLM 코딩 실수 관찰을 기반으로 구성했다.

> 이 원칙들은 속도보다 신중함을 우선한다. 사소한 작업에서는 판단에 맡기라.

## 상황별 원칙 매핑

| 실행 상황 | 적용 원칙 | 파일 |
|-----------|-----------|------|
| 요구사항 파악, 모호한 지시 수신 | 먼저 생각하기 | @think-before-coding.md |
| 새 코드 작성, 기능 구현 | 단순함 우선 | @simplicity-first.md |
| 기존 코드 수정, 버그 수정 | 정밀 변경 | @surgical-changes.md |
| 작업 완료, 다단계 작업, 검증 | 목표 중심 실행 | @goal-driven-execution.md |

## 문제 → 원칙 → 해결

| 문제 | 원칙 | 해결 |
|------|------|------|
| 가정을 확인하지 않고 구현 | 먼저 생각하기 | 가정을 명시하고, 불확실하면 질문 |
| 불필요한 추상화와 과설계 | 단순함 우선 | 요청한 것만, 최소 코드로 |
| 요청과 무관한 코드 변경 | 정밀 변경 | 요청에 해당하는 줄만 수정 |
| 성공 기준 없이 작업 | 목표 중심 실행 | 검증 가능한 기준 정의 후 루프 |

## 효과 검증 체크리스트

```
[ ] diff에 요청과 무관한 변경이 없는가
[ ] 구현 전에 명확화 질문이 먼저 오는가
[ ] PR이 최소한의 변경만 담고 있는가
[ ] 불필요한 추상화 없이 단순한 코드인가
[ ] 과설계로 인한 재작업이 줄었는가
[ ] 각 변경의 성공 기준이 사전에 정의되었는가
[ ] 기존 코드 스타일이 일관되게 유지되는가
```
